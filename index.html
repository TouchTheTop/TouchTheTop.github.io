如何使用cross-env配置多个环境（前端)

通过js在平台设置不同的环境变量的工具

第一步：安装

npm install --save-dev cross-env

第二步：指令

在package.json中的scripts加上一条,cross-env env=production webpack-dev-server --info=false --host 0.0.0.0 --hot --config ./build/webpack.dev.config.js

第三步:完成

让我们回到第二步，执行指令吧！

TIP

我们可以写多个配置文件，引入这个环境变量，为啥要写多个配置文件呢？那是因为不同环境的配置需求是不一样的，例如，测试环境有 A 系列域名，开发环境有 B 系列域名等。
